# Contributing

## Working principles
- Keep changes small and focused
- Prefer clarity over completeness
- Avoid premature formalization
- Document decisions explicitly when they affect the model or workflow
- This repo is framework-agnostic

## Document creation
- New documents should be added under `docs/`
- Use kebab-case for file names
- Prefer one topic per document
- Link related documents rather than merging unrelated concepts into one file

## Model artifacts
- Early contract examples should live under `model/contracts/`
- Until a formal schema exists, examples may be written as Markdown
- Do not treat examples as final standards unless explicitly marked

## Changes and review
- Use GitHub Issues for work items
- Use PRs for substantial changes
- Prefer descriptive titles that reflect the problem or decision
